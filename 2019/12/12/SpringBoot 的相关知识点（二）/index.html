    • Spring Boot 的相关知识点（二）

    1 配置文件的加载位置

    Spring Boot 在启动的时候会去扫描以下几个位置的 application.properties 或者 application.yml 文件作为配置文件：

    • file: ./config （整个项目根路径）
    • file: ./
    • classpath: /config/ （resource 文件夹）
    • classpath:

    四个位置是按照优先级排序的，若以上几个位置都存在同配置属性，则上面位置的文件会覆盖下面位置。

    Spring Boot 会全部加载四个不同位置的配置文件，这样可以实现互补配置，进行分工。

    对于默认位置的修改，可以通过修改 spring.config.location 来完成。

    项目打包好后，还可以通过命令行参数的方式，且这种方式优先级最高。

    2 配置文件自动配置的原理

    配置文件中所能配置的属性见 官方文档

    自动配置的原理：

    1. Spring Boot 启动的时候加载主配置类，开启了自动配置功能 @EnableAutoConfiguration

    2. @EnableAutoConfiguration 的作用：

      将类路径下 META-INF/spring.factories 里面配置的所有 EnableConfiguration 的值加入到了容器中

    3. 每一个自动配置类 xxxxAutoConfiguration 来进行自动配置功能

    4. 所有在配置文件中能配置的属性都在 xxxxProperties 类中封装着，配置文件所能配置的属性需要参照某个功能所对应的这个属性类

      [ 细节知识点 ]

    @ConditionalOnXxxx 注解的作用：只有 @Conditional 指定的条件成立，才可以给容器中添加组件，配置文件里面的内容才生效。
